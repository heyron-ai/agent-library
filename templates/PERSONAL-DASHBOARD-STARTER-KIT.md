# PERSONAL-DASHBOARD-STARTER-KIT.md

Build your own dashboard with your agent.

This is a **starter brief + build plan** you can hand to your agent if you want a simple personal dashboard.

No coding knowledge required. Just copy this into chat and tell your agent to build it step by step.

---

## What this is for

Use this if you want a dashboard for things like:

- your goals
- tasks
- calendar
- habits
- content ideas
- leads
- notes
- reminders
- agent outputs
- project progress

This is for users who want something visual and useful — not a giant enterprise app.

---

## The simplest version

Tell your agent:

```text
I want you to build me a simple personal dashboard.

Requirements:
- Make it clean, simple, and visual
- Use a dark theme
- Make it mobile-friendly
- Start with fake/sample data first so I can see it working
- Then help me replace sample data with my real data

Sections I want:
- Today's priorities
- This week's goals
- Task list
- Calendar / upcoming events
- Notes or quick capture
- Progress tracker

Keep version 1 simple. Do not overbuild it.
```

## Best prompt to give your agent

```text
Build me a simple dashboard I can actually use.

Start with a version that works locally in the browser as a simple HTML/CSS/JavaScript app.

Rules:
- Keep it beginner-friendly
- Make it look polished
- Do not add unnecessary complexity
- Use sample data first
- Explain where the data lives
- Make it easy for me to edit later
- Give me a version 1 before adding fancy features

My dashboard type is: [personal command center / business dashboard / content dashboard / project dashboard]

The sections I want are:
- [list your sections]

The vibe I want is:
- [minimal / clean / dark / modern / playful]

After version 1 works, help me improve it step by step.
```

---

## Smart build order

Tell your agent to build in this order:

### Version 1
- one clean page
- header
- 4 to 6 useful cards
- sample data
- working layout

### Version 2
- editable data file
- better styling
- real categories
- better mobile layout

### Version 3
- charts
- filters
- status colors
- saved notes
- optional integrations

This keeps the project from turning into chaos.

## Where your data can live

Ask your agent to choose one of these simple options:

### Option 1: hardcoded sample data
Best for first draft

### Option 2: local JSON file
Best for simple dashboards you want to edit easily

### Option 3: Airtable / Notion / Google Sheets later
Best once version 1 already works

Important:
**Do not start with 5 integrations unless you actually need them.**
Get a simple version working first.

---

## What to tell your agent if it overcomplicates things

```text
This is getting too complicated.
Please simplify it.
I want a clean version 1 that works, not a giant app.
Cut anything unnecessary and keep the core dashboard useful.
```

---

## What to tell your agent if you want it prettier

```text
Improve the visual design of this dashboard.
Keep the same structure, but make it feel more polished, modern, and premium.
Use spacing, card styling, typography, and color well.
Do not redesign the whole app from scratch.
```

---

## What to tell your agent if you want it published

```text
Now help me publish this dashboard so I can open it from a link.
Use the easiest option available and explain each step simply.
```

---

## The main rule

**A working simple dashboard is better than an ambitious broken dashboard.**

Version 1 should feel like:
- useful
- clean
- understandable
- easy to improve

Not like:
- a startup trying to raise Series A

---

## Success test

Your dashboard is good enough if:
- you can open it easily
- it shows useful information
- you understand what each section does
- your agent can improve it without rebuilding everything
