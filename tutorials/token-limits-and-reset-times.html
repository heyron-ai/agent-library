Where Did My Tokens Go?

A fast, no-jargon way to understand usage limits, reset timing, and why "I paid already" can still hit provider caps.

Common #help question: "I’m burning through tokens. Do we have a limit, and when does it reset?" This guide gives you a deterministic way to diagnose that in under 5 minutes.

First: there are usually two different limit systems

  • Platform-side limits (your heyron plan/session/runtime behavior).
  • Provider-side limits (OpenRouter/OpenAI/etc key credits, quota, rate limits, or monthly caps).

If either side is exhausted or blocked, your run can fail.

Reality check: a heyron subscription does not automatically grant unlimited third-party provider usage when your workflow depends on your own external API key.

2-minute diagnosis flow

  1. Capture the exact error text (don’t paraphrase).
  2. If it mentions 403, quota, credits, limit exceeded, or rate limit, suspect provider-side cap first.
  3. Check whether this run used your own provider key or heyron-included defaults.
  4. Start a fresh session and run one tiny prompt to confirm if failure is persistent or thread-specific.

How reset timing usually works

  • Provider keys: reset windows are controlled by that provider (monthly billing cycle, top-up balance, or per-minute rate windows).
  • Rate limits: can reset in seconds/minutes, not billing-cycle boundaries.
  • Plan usage buckets: may reset on your heyron billing cadence or configured allowance window.

Avoid this trap: waiting for midnight in your local timezone may do nothing. Reset windows often follow provider account timezone/UTC or billing anniversary date.

Copy/paste verification prompt

I need a usage-limit diagnosis with proof. 1) Tell me which key/provider this session is using. 2) Quote the exact last error text verbatim. 3) Classify this as: provider quota, provider rate limit, invalid key/model, or platform/session issue. 4) Give me the next safe action in one step. 5) If you can’t determine reset timing, tell me exactly which billing/usage page I should check.

When to escalate to #help

  • You verified key + model are valid, but usage errors continue across fresh sessions.
  • Reset window should have passed, but the same quota error persists.
  • Error category changes randomly between runs (for example, quota + auth + model-not-found).

Support packet (paste this)

Token / limit issue report Exact error text: Provider/model used: Using personal provider key? (yes/no): Fresh session retest done? (yes/no): Same error after retest? (yes/no): When you expected reset (date/time + timezone): Any recent key/model changes: Screenshot of usage/quota page (optional):

Bottom line: treat token issues as a routing problem first: identify which limit system failed, then verify the correct reset window for that system.

Once you separate provider caps from platform usage, these issues become predictable and fast to fix.
