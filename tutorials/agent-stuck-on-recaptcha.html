Troubleshooting

Agent Stuck on reCAPTCHA or Cloudflare Check?

This is a normal anti-bot checkpoint. Your agent isn’t broken—you just need a human handoff for this step.

A recurring community pain point: "My agent can browse, but it gets blocked when a site asks for reCAPTCHA / Verify you’re human." That’s expected behavior on many websites.

Reality check: agents can do a lot of browser work, but anti-bot challenges are intentionally designed to stop automation. Treat this as a manual checkpoint, not a full failure.

What’s happening

  • Sites like Google, Cloudflare-protected apps, ticketing tools, and marketplaces use bot defenses.
  • When triggered, the page can require mouse behavior, puzzle solving, or identity/session trust checks.
  • Your agent may read page text but still fail at final protected actions.

Fast diagnosis (60 seconds)

  1. Ask your agent to quote the exact blocker text from the page.
  2. If you see "Verify you are human," "reCAPTCHA," or "Checking your browser," assume manual intervention is required.
  3. Have the agent save progress and list the next 1–3 actions it was about to take.

Known-good workflow

  1. Agent prepares everything up to the checkpoint (fields, draft text, selected options).
  2. You complete the CAPTCHA/challenge manually.
  3. Agent resumes from the now-authenticated page/session and continues remaining steps.

Avoid this trap: repeatedly retrying the same blocked action can trigger stricter anti-bot defenses. Pause, do one manual checkpoint, then resume.

Copy/paste recovery prompt

You hit an anti-bot checkpoint. Do this now: 1) Tell me the exact page state and blocker message. 2) Save all progress made so far. 3) Give me the one manual step I should do next. 4) After I confirm it's done, continue automatically with the remaining steps. 5) If blocked again, switch to a fallback plan and tell me exactly what changed.

Fallback plans when CAPTCHA repeats

  • Switch workflow: have the agent generate content/data and do final submit manually.
  • Use official APIs/connectors: where available, API-based flows are more stable than UI-click automation.
  • Reduce suspicious patterns: fewer rapid retries, fewer tab hops, and no repeated failed login loops.

When to escalate to #help

  • Manual challenge passed, but the same page keeps challenging every action.
  • Agent claims completion but no action was actually submitted.
  • The issue started after a runtime/config change and now happens everywhere.

Support packet

CAPTCHA / Anti-bot issue report Target site: Task attempted: Exact blocker text: What worked before blocker: Manual step completed? (yes/no): What happened after manual step: Screenshots (optional):

Bottom line: CAPTCHA barriers are expected. The winning pattern is agent-does-the-work + human-clears-checkpoint + agent-resumes.

This one change in workflow removes a lot of frustration and keeps your project moving.
