Module 2: Agent Shows “Error” and Won’t Reply?

A fast isolation flow for beginners when every agent reply fails during setup.

If you're in early onboarding (often Module 2) and your agent keeps returning an error instead of a normal reply, don't keep random retrying. Use this 4-minute flow.

What we’re seeing in community right now

New users are hitting repeated "agent error / no reply" loops, then getting stuck because they don’t know whether it’s setup, runtime health, or support-routing.

Step 1) Capture One Clean Failure

  1. Open a fresh chat/session.
  2. Send one tiny prompt: Reply with exactly: READY.
  3. Copy the exact error line (or screenshot it).

Step 2) Rule Out Surface/Context Confusion

  • Retry the same tiny prompt in one other surface (for example, dashboard chat vs Discord).
  • If one surface works and another fails, this is usually context/routing/policy, not model quality.
  • If both fail with the same error, continue to runtime-health checks.

Step 3) Check for Runtime Health Pattern

If your errors mention things like runtime not healthy, health 502, or Gateway tool invoke failed: 500, you're likely dealing with runtime-side instability, not prompt quality.

Loop-break rule

Do one clean retry after any restart/reconnect change. If the same error returns, stop retry loops and escalate with evidence.

High-Friction Cases

“How do I create a help ticket?”

Use: https://heyron.ai/support. Include exact error text and one screenshot. Don’t post only “it doesn’t work.”

Community says “start over” — should I?

Only after you capture one clean failure and one tiny canary test. Starting over blindly can erase useful evidence and slow support.

The error happens in every reply

That usually points to runtime/config health, not bad prompting. Use the packet above and escalate once with full evidence.

Security note

Never paste full tokens/API keys in tickets or Discord screenshots. Redact secrets before sharing.
