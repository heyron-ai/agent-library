Troubleshooting

Did I Create a Second Agent by Accident?

A fast way to diagnose account/workspace mix-ups when your bot name, memory, or settings suddenly look wrong.

This is more common than people think: you switch devices, click a different invite link, or log in with another Google account—and now it feels like your original agent disappeared.

Good news: in most cases, your original agent still exists. You are just in a different account, workspace, or surface.

Common signs of a second-agent/account mix-up

  • Your agent name/personality looks generic again.
  • Files like SOUL.md or MEMORY.md look missing or blank.
  • Discord DM behaves differently from dashboard chat.
  • You see a different workspace path than expected.
  • You are sure you configured things before, but nothing matches now.

2-minute verification flow

  1. Confirm account identity: verify the email/account currently logged in on dashboard and Discord-related tools.
  2. Check workspace identity: confirm the active workspace/project name matches your expected one.
  3. Run an identity prompt: ask the agent to quote name + key file paths it is using.
  4. Compare to your known baseline: old screenshots, notes, or expected file names.

Copy/paste identity check prompt

Before doing anything else, print: 1) Your agent name 2) Current workspace/root path 3) Whether SOUL.md exists (include exact path) 4) Whether MEMORY.md exists (include exact path) 5) The current timezone 6) The exact account/workspace label you think this session belongs to If anything is missing, stop and ask me one clarifying question before continuing.

If DM and dashboard feel like different bots

  • Same identity, different context: DM, server channels, and dashboard chats can be separate conversation threads.
  • Not automatically shared: prior chat context from one surface may not appear in another.
  • Use a handoff: paste a compact context summary when switching surfaces.

Important: this is usually a context/account mismatch, not memory corruption.

Quick recovery options

  1. Log out and log back in with the account that originally created your agent.
  2. Open the correct workspace/project explicitly.
  3. Run the identity check prompt above again.
  4. If still mismatched, start a fresh session and pass a short handoff (goal + current files + next action).

High-friction mistakes to avoid

  • Assuming browser profile = correct account: many users are signed into multiple Google/Discord profiles.
  • Comparing against another user’s screenshot: names/labels may differ by account and setup.
  • Continuing work before identity check: this creates more confusion and duplicate files.

Do not delete anything yet. First confirm you are in the right account/workspace. Deleting the “wrong” one can remove useful clues for recovery.

Escalation packet for #help

Possible second-agent/account mix-up Expected agent/workspace: Current visible agent/workspace: What changed recently (device/login/link): Identity check output (paste): What I expected instead: Screenshots available: yes/no

Most “my bot changed” incidents are fixable in minutes with identity + workspace verification. Verify first, then troubleshoot behavior.
