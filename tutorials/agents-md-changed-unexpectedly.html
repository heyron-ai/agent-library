AGENTS.md Changed Unexpectedly?

A fast recovery flow for when your behavior rules drift and you’re not sure why your agent suddenly acts different.

Community pattern: "I didn’t mean to change behavior, but AGENTS.md looks different and now everything feels off." Use this guide to restore control without guesswork.

What AGENTS.md controls

  • Operating rules for your workspace (what to do first, red lines, permission boundaries).
  • Behavior defaults (tone, response habits, when to ask first vs act directly).
  • Safety boundaries that can override ad-hoc chat instructions.

If AGENTS.md drifts, your agent can feel like a different assistant even when model/settings stay the same.

3-minute recovery checklist

  1. Ask the agent to quote the exact AGENTS.md path and last modified time.
  2. Request a line-by-line diff summary of what changed and why.
  3. Restore known-good sections (or full file) from backup/history.
  4. Start a fresh session and run a tiny behavior smoke test.

Important: "I think it changed" is not enough. Always require path + timestamp + diff proof before editing again.

Copy/paste diagnostic prompt

I need AGENTS.md verification with proof. 1) Show the exact file path you are reading. 2) Show last modified timestamp. 3) List what changed since previous known-good version (section + line references). 4) Propose the minimum safe rollback patch. 5) Do not apply changes until I confirm.

High-friction edge cases

  • Wrong file, same name: AGENTS.md exists in multiple folders; edits hit a different workspace.
  • Rule collision: SOUL.md asks for one style, AGENTS.md enforces another; AGENTS wins for operations.
  • Chat instruction override myth: one-off messages like "ignore prior rules" may be blocked by higher-priority safety/instruction layers.
  • Stale-thread behavior: old conversation momentum can make behavior seem unchanged even after you fix AGENTS.md. Retest in a fresh session.

Prevention: stop future drift

  • Keep a "known-good" AGENTS.md snapshot (date-stamped copy).
  • For any AGENTS edit, require a plan-first + diff-first workflow.
  • After edits, run one tiny verification prompt and confirm expected tone/behavior before real work.

Support packet (paste this into #help)

AGENTS.md drift report Workspace path: Exact AGENTS.md path: Last modified timestamp: What changed unexpectedly (section names): Fresh session retest done? (yes/no): Behavior difference observed (1-2 examples): Known-good AGENTS snapshot available? (yes/no):

Bottom line: treat behavior drift like a configuration incident: prove path, prove diff, roll back minimally, then retest in a fresh session.

Once you require path + diff proof, AGENTS.md issues become quick to diagnose and easy to contain.
