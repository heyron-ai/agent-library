Agent Can’t Read My PDF?

If your PDF is ignored, summarized incorrectly, or returns “I can’t access this,” the problem is usually format/visibility—not your model.

Fast reality check

Most PDF failures come from one of five causes:

  • the file was never actually uploaded to the active chat/thread
  • the PDF is image-only (scanned) with no selectable text
  • the file is password-protected or restricted
  • the wrong version of the file is being analyzed
  • you shared a private link the agent can’t open

2-minute diagnosis flow

  1. Prove file visibility first.
    Prompt: Before summarizing, confirm the exact filename you can see and quote the first 120 characters from page 1.
  2. Check if text is selectable.
    Open the PDF yourself and try to highlight/copy text. If you can’t, it’s likely a scanned image PDF and needs OCR.
  3. Rule out private-link mismatch.
    If sharing by URL, test with a truly public link or upload the file directly into the chat.
  4. Verify version identity.
    Rename the file with a version suffix like proposal-v3.pdf and re-upload to avoid stale-file confusion.

If the agent can quote page text correctly, extraction works. Then you can trust the summary task.

High-friction mistakes

  • “I pasted a Drive link, so it can read it”: many links are login-gated.
  • Uploading once, then switching threads: file attachments do not always follow you between contexts.
  • Using scanned contracts/invoices without OCR: image PDFs often require text extraction first.
  • Asking for deep analysis before visibility proof: always require quote-proof first.

Copy/paste recovery prompt

Do not summarize yet. First, confirm the exact PDF filename you can access, then quote 2 short excerpts: one from near the beginning and one from the middle (include page numbers). If you cannot do that, explain exactly why (missing file, image-only PDF, permissions, or other) and suggest the minimum fix.

When to escalate to #help

Escalate with this packet:

  • Timestamp + timezone
  • Where the file was provided (upload vs URL, and which thread/channel)
  • Whether text is selectable in the PDF
  • Exact error text (verbatim) or failed quote-proof result

Without quote-proof, support can’t quickly tell whether this is access, OCR, or context mismatch.

Bottom line: require filename + quote-proof before any summary. That one step catches most PDF issues in under 2 minutes.
