Cron Job Created but Not Showing Up?

If your agent says it created a cron reminder but you can’t find it in the dashboard, this guide isolates the mismatch in under 2 minutes.

Fast reality check

Most of the time this is not a scheduler bug. It’s one of these:

  • Job was created in a different session/thread.
  • Job was created with a different session target than expected.
  • The agent described creation but never actually ran the write call.

2-minute diagnosis flow

  1. Ask for proof, not a summary.
    Prompt: Show the exact cron job ID you created and paste the full job object.
  2. Confirm where it was created.
    Prompt: Tell me the session target and destination channel/thread for this job.
  3. List jobs immediately.
    Prompt: List all active cron jobs now and include disabled jobs too.
  4. Run a canary test.
    Create a one-shot reminder 2 minutes out, with unique text like CANARY-APR03-1023.

If the canary appears correctly, your scheduler works. The original issue was route/session mismatch.

Known high-friction mistakes

  • “Agent cronjobs” vs “Control cronjobs” confusion: jobs may show in one area depending on how they were created.
  • Old thread vs new thread: same agent identity, different session context.
  • Ambiguous reminder request: no destination specified, so output lands somewhere unexpected.
  • Silent failure after config edits: runtime restart needed before new behavior is visible.

Copy/paste recovery prompt

Use this exactly:

Create a one-time reminder for 2 minutes from now. Before creating it, state the exact destination (sessionTarget + channel/thread). After creating it, return the cron job ID and then list cron jobs to prove it exists. Use reminder text: CANARY-CRON-VISIBILITY-TEST.

When to escalate to #help

Escalate only after you have:

  • Timestamp + timezone
  • Expected destination vs actual destination
  • Cron job ID (or explicit failure to return one)
  • Canary reminder result

Without this evidence, support has to guess—and resolution takes much longer.

Bottom line: prove job ID + destination + canary delivery. That closes most “it said it made the cron job but I can’t see it” cases fast.
